# 后续功能与优化路线图

本文档记录 Live Serial Plotter 后续可做的 feature、性能优化和工程任务。它是任务池和方向参考，不代表固定发布时间表。

## 产品定位

Live Serial Plotter 优先做成 VS Code 内的轻量实时串口可视化工具，服务嵌入式、传感器、机器人、Arduino、ESP32、STM32 和固件调试场景。

短期目标不是完整串口工作台，而是把“连接串口、解析文本、看原始日志、看实时曲线、保存和复盘数据”这条链路做稳、做快、做清楚。

## 阶段 1：MVP 稳定化

目标：让当前功能在真实使用中可靠，不轻易因为串口异常、高频输出或打包问题失效。

### 连接与串口会话

- [ ] 连接中、断开中状态：避免用户重复点击导致状态错乱。
- [ ] 串口异常提示细化：区分端口不存在、权限不足、端口被占用、native binding 加载失败。
- [ ] 自动处理设备拔出：断开连接、保留日志、提示用户刷新端口。
- [ ] 连接参数持久化：端口、波特率、parser mode 已有基础状态，后续可补数据位、停止位、校验位。
- [ ] 发送文本支持行尾选项：none、LF、CRLF、CR。

### Webview 基础体验

- [ ] 图表暂停/继续：暂停时继续接收并缓存或明确丢弃，行为需要在 UI 上可见。
- [ ] 清空图表：独立于清空 raw log。
- [ ] 自动滚动开关：raw log 用户手动上翻时不强制滚到底。
- [ ] 错误 toast 排队或覆盖策略：避免连续错误时提示闪烁或丢失关键信息。
- [ ] 空状态优化：无端口、未连接、等待数据、parser 无数字输出时有明确状态。

### 性能和稳定性

- [ ] raw log 保持最大行数限制，并避免每行都整表重建 DOM。
- [ ] 图表数据继续保持最大点数限制，验证长时间运行不会持续涨内存。
- [ ] 高频串口输入下批量刷新图表，避免每个点触发一次 `setData()`。
- [ ] 用模拟串口压测 1 Hz、10 Hz、100 Hz、1000 Hz 输入频率下的 UI 响应。

### 测试

- [ ] 增加 Webview panel 消息路由测试：端口列表、连接状态、错误、数据批处理。
- [ ] 增加连接失败和断开清理单元测试。
- [ ] 增加 parser 自动模式边界测试：空行、非数字字段、混合格式、非法 JSON。
- [ ] 固化 `pnpm package` 的 smoke test 检查项，重点验证 native binding 是否被打进 VSIX。

## 阶段 2：实时绘图能力增强

目标：让图表从“能看”变成“调试时真的好用”。

### 图表交互

- [ ] 鼠标游标读数：显示当前时间点各通道数值。
- [ ] 缩放和平移：支持查看历史区间，并提供回到实时模式。
- [ ] Y 轴自动缩放开关：实时模式自动适配，手动模式保持用户缩放。
- [ ] 通道颜色配置：用户可修改每个通道颜色。
- [ ] 通道排序和隐藏：图例中可隐藏通道，后续可支持固定顺序。
- [ ] 单位显示：通道可配置单位，例如 `temp (degC)`、`rpm`、`voltage (V)`。

### 数据结构和 uPlot 约束

- [ ] 新增或删除通道时才重建 uPlot。
- [ ] 普通数据追加只更新数据数组并调用 `setData()`。
- [ ] 通道显示切换只调用 `setSeries()`。
- [ ] 尺寸变化只调用 `setSize()`。
- [ ] 图表数据数组保持非响应式、可原地更新，避免引入框架后走深层响应式。
- [ ] 对高频数据采用批处理或 `requestAnimationFrame` 合并刷新。

### 多通道和大数据

- [ ] 支持显示当前采样率和点数。
- [ ] 支持每通道最近值、最小值、最大值、平均值。
- [ ] 支持按时间窗口保留数据，例如最近 30 秒、5 分钟、固定 3000 点。
- [ ] 评估是否需要 downsampling；当点数显著超过屏幕像素时再考虑。

## 阶段 3：解析器、预设和设备配置

目标：降低用户接入设备的成本，让常见串口输出可以快速变成可视化曲线。

### Parser 能力

- [ ] CSV header 支持：首行 `time,temp,hum` 可作为通道名。
- [ ] CSV 无 header 模式支持自定义通道名。
- [ ] key-value 支持更多分隔符：空格、逗号、分号混合场景。
- [ ] JSON Lines 支持嵌套字段扁平化，例如 `imu.ax`、`imu.ay`。
- [ ] auto parser 给出识别结果提示，方便用户知道当前使用的格式。
- [ ] parser 无法提取数字时显示原因和示例。

### 配置预设

- [ ] 设备 profile：端口参数、parser、通道名、单位、颜色、行尾发送格式。
- [ ] profile 导入/导出：JSON 文件或 VS Code global state。
- [ ] 最近使用 profile 列表。
- [ ] 为内置模拟串口提供示例 profile。

### 用户自定义解析

- [ ] 先评估安全边界，不在 Webview 中执行用户脚本。
- [ ] 可以考虑安全 DSL 或有限表达式，例如字段映射和正则提取。
- [ ] 自定义 parser 必须可测试、可导入导出，并且不能破坏 CSP。

## 阶段 4：记录、导出和回放

目标：把实时调试扩展成可复盘的数据工具。

### 数据记录

- [ ] 原始串口日志导出为 `.log` 或 `.txt`。
- [ ] 解析后的数据导出为 CSV。
- [ ] JSON Lines 输入可保留原始 JSONL。
- [ ] 记录开始/停止按钮，显示记录时长和文件大小。
- [ ] 明确记录的是 raw line、parsed sample，还是两者都记录。

### 回放

- [ ] 从 CSV/JSONL/log 回放数据到图表。
- [ ] 支持回放速度：0.5x、1x、2x、尽快。
- [ ] 回放时复用当前 parser 或使用文件中的 metadata。
- [ ] 回放模式和真实串口模式在 UI 上明确区分。

### 文件和工作区集成

- [ ] 默认保存到用户选择的位置，不自动污染工作区。
- [ ] 可选保存到当前 workspace 的 `.live-serial-plotter/`。
- [ ] 导出文件附带 metadata：时间、端口、波特率、parser、通道信息。

## 阶段 5：命令发送和调试工作流

目标：提升串口交互效率，服务固件命令行、AT 指令、设备控制等场景。

- [ ] 发送历史：上下键或下拉列表复用历史命令。
- [ ] 常用命令按钮：profile 里保存命令名和内容。
- [ ] 行尾模式：none、LF、CRLF、CR。
- [ ] 十六进制显示和发送：作为高级模式，默认隐藏。
- [ ] 可选时间戳显示：raw log 每行显示本地时间或相对时间。
- [ ] 支持复制选中日志、清空日志、保存日志。

## 阶段 6：架构演进和 Vue 3 评估

目标：在功能复杂度上升时保持可维护性，但不为了框架而重写。

### 先做 Vanilla TypeScript 模块化

- [ ] 拆分 Webview bridge：封装 `acquireVsCodeApi()`、`postMessage()`、状态持久化。
- [ ] 拆分 controls：端口、波特率、parser、连接按钮和发送表单。
- [ ] 拆分 plot：uPlot 实例、数据数组、legend、resize、series 更新。
- [ ] 拆分 raw log：日志缓冲、滚动策略、清空和导出。
- [ ] 拆分 toast/status：错误提示和连接状态展示。

### 触发 Vue 3 迁移的条件

满足以下多个条件时再考虑迁移：

- UI 出现多个复杂面板，例如 profile、记录、回放、通道配置、命令面板。
- Webview 入口文件再次增长到难以维护，并且模块化后仍不够清晰。
- 表单、列表、条件渲染和组件复用成为主要开发成本。
- 需要更系统的组件测试和 UI 状态组织。

### Vue 3 迁移原则

- [ ] 只迁移 Webview UI 层，不重写 Extension Host、parser、serial service 和 shared protocol。
- [ ] 使用 Vue 3 + Composition API + Single File Components。
- [ ] Vue 管理低频 UI 状态：端口、连接状态、parser、profile、toast、通道可见性。
- [ ] uPlot 实例和大批量图表数据保持非响应式，例如使用 `shallowRef`、`markRaw` 或模块内普通变量。
- [ ] 高频数据流不直接驱动组件整树渲染。
- [ ] 迁移前先补齐当前 Webview 行为测试，迁移后用同一组测试验证无回归。

## 阶段 7：测试、CI 和发布质量

目标：降低 native binding、跨平台打包和串口行为的不确定性。

- [ ] CI 覆盖 `pnpm check` 和 `pnpm package`。
- [ ] release workflow 保持分平台 VSIX 打包。
- [ ] 增加 VSIX 内容检查：确认 `dist/node_modules/@serialport/bindings-cpp` 存在。
- [ ] 增加 Extension Development Host smoke test 指南。
- [ ] 维护模拟串口和虚拟串口 E2E 场景。
- [ ] 对 Linux、macOS、Windows 分别记录常见串口权限问题和排查方法。

## 优先级建议

建议按下面顺序推进：

1. 稳定连接、断开、错误提示和 raw log 行为。
2. 加图表暂停、清空图表、自动滚动开关。
3. 加记录和 CSV 导出，因为这会明显提升实际调试价值。
4. 加图表游标、缩放和通道配置。
5. 加 profile 和 parser preset。
6. 再评估是否迁移 Vue 3。

## 暂不优先的方向

- 完整 Modbus、CAN、I2C、SPI 工作台。
- Web 版 VS Code 支持。
- 在 Webview 中执行用户自定义 JavaScript。
- 大型 UI 组件库。
- 在首版引入 React 或 Vue 3 全量重写。

这些方向不是不能做，而是会显著扩大范围。除非产品目标改变，否则应等核心串口可视化链路稳定后再评估。
